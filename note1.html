Variables & Data Types

In JavaScript, variables are used to store data values. These values can then be accessed or manipulated throughout the program.

What is a Variable?

A variable is a named storage location in memory that holds data which can change throughout the course of a program. Variables allow developers to store and reference values like numbers, strings, and more complex data structures.

Declaring Variables

In JavaScript, variables can be declared using three keywords: var, let, and const.

  • let: Used to declare a variable that can be reassigned later. It has block scope.
  • const: Used to declare a constant variable that cannot be reassigned once set. It has block scope.
  • var: Used to declare a variable with function scope. It is considered outdated, and it's best to use let or const.

Data Types

In JavaScript, there are several types of data you can store in variables. These types are categorized as primitive and non-primitive.

Primitive Data Types

  • String: Represents a sequence of characters. Example: "Hello".
  • Number: Represents both integers and floating-point numbers. Example: 25, 3.14.
  • Boolean: Represents either true or false.
  • Null: Represents the intentional absence of any value. Example: null.
  • Undefined: Represents a variable that has been declared but not assigned a value. Example: let x;.
  • Symbol: Represents a unique and immutable value, often used as an identifier for object properties.

Non-Primitive Data Types

  • Object: Represents a collection of key-value pairs. Example: { name: "Alice", age: 25 }.
  • Array: Represents a list of values stored in an ordered collection. Example: [1, 2, 3].

Variable Naming Rules

In JavaScript, there are specific rules and best practices to follow when naming variables. Here are the 6 key rules for naming variables:

  • 1. Use camelCase for multi-word variable names: Start with a lowercase letter and capitalize subsequent words. Example: userName, totalAmount.
  • 2. Variable names must begin with a letter, underscore (_), or dollar sign ($). Example: _user, $price.
  • 3. No spaces allowed in variable names. You cannot name variables like user name â€” use userName instead.
  • 4. Avoid using reserved keywords as variable names. Words like class, function, return, and others cannot be used as variable names.
  • 5. Variable names are case-sensitive. userName and username would be considered two different variables.
  • 6. Use meaningful names for your variables. Instead of naming a variable x or temp, name it something descriptive like userAge or orderTotal.

Example: Naming Variables Correctly

let userName = "Alice";
let userAge = 25;
const totalAmount = 100.50;

Type Conversion

JavaScript allows you to convert values from one type to another. This is called type conversion. Some common conversions include:

  • String() : Converts a value to a string.
  • Number() : Converts a value to a number.
  • Boolean() : Converts a value to a boolean.

Example: Type Conversion

let num = 42;
let str = String(num);  // "42" (string)
let bool = Boolean(0);  // false (boolean)

Conclusion

Variables are essential in JavaScript for storing and manipulating data. When naming variables, always follow the rules of good naming conventions to make your code more readable and maintainable. Also, understand the data types available in JavaScript so you can store and work with different kinds of information.
